The story so far

DRAFT CONTENT

The aim of the test and learn work is to build the capabilities required for a Geospatial Register. This will be acheived by transforming the data in a 5K sample area of Bristol.

There are 6 key areas of the work which support this aim

  • Create data from raster title plans
  • Convert vector (TMF) data to an open data format
  • Match textual and spatial register data
  • Extract text from deeds
  • Join text from deeds to the combined textual and spatial data
  • Visualise the result

To validate the results in the Bristol 5K, the team are also testing methods using data from 3 local authorities: Gloucester, Manchester and Bournemouth.

So what are Test and Learn doing?

Test and Learn is testing methods to build the capabilities to build a geospatially enabled register. Much like building a house, there's a lot that needs to be done before you can start building. This includes getting the right tools in place, figuring out the right materials to use and understanding which methods will work: Test and Learn is working in all of these areas to understand how HMLR can create those building blocks.

First digging for the clay, identifying existing problem areas, looking at the TMF, considering extraction from deeds etc.

The team then tested ways of transforming those raw materials into something new and useable.

These methods were tested to check how well they stand up. Like firing bricks, this helps us to see where issues might lie and get a good sense of what works, pretty much testing each method to breaking point.

And from the successful methods, we were able to visualise that result, laying the foundations for the ambition of a geospatial register.

Test and learn isn't building the house, but providing methods for making house building possible - from there the Geospatial Register could still take many forms.

Test and Learn is not about building a service or building the register, but instead figuring out how it might be possible. The answers are not all there, and there is no case study which takes into account the myriad of complexities in HMLR's data and practices.

This work is therefore vital to support the future ambitions of the business and move forward from a sure foundation and understanding. The work from the Test and Learn team represents a signifcant improvement to our knowledge about the register and the methods for modernizing it.
